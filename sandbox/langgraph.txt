1. OVERVIEW OF AI AGENTIC FRAMEWORKS IN 2025
================================================================================

Agentic AI frameworks enable building systems that can reason, plan, use tools,
and act autonomously. The landscape has evolved rapidly, and in 2025-2026 the
key frameworks are:

  • LangGraph (by LangChain) — Graph-based, stateful, production-grade
  • CrewAI — Multi-agent orchestration with role-based agents
  • Microsoft AutoGen — Multi-agent conversations, event-driven
  • OpenAI Agents SDK — Official OpenAI agent framework (replaces Swarm)
  • Google ADK — Google's open-source agent framework
  • Smolagents (Hugging Face) — Lightweight, code-agent approach
  • Mastra — TypeScript-first agent framework
  • Pydantic AI — Type-safe agent framework
  • Agno — Lightweight multi-modal agent framework

Key Trends:
  - Shift from linear chains to cyclical, graph-based workflows
  - State management and persistent memory are now first-class concerns
  - Multi-agent orchestration (agent-to-agent communication)
  - Tool use and function calling as standard primitives
  - Production features: streaming, checkpointing, observability

3. LANGGRAPH DEEP DIVE — CORE CONCEPTS
================================================================================

LangGraph is a graph-based framework by LangChain for building stateful,
multi-step AI agent workflows. It is designed for production use with features
like streaming, checkpointing, and human-in-the-loop.

--- 3.1 The Four Pillars ---

  A. STATE
     - The shared data object that flows through the entire graph.
     - Defined using Python's TypedDict or Pydantic models.
     - Each node reads from and writes updates to the state.
     - Managed through channels that define how state is updated.

  B. NODES
     - Individual processing steps in the graph.
     - Can be: Python functions, LLM invocations, tool calls, or sub-graphs.
     - Each node receives the current state and returns updates to it.

  C. EDGES
     - Define the control flow between nodes.
     - Types:
       • Normal edges — always go from node A to node B.
       • Conditional edges — route based on the state/output of a node.
       • Entry/exit points — START and END special nodes.

  D. GRAPH
     - The overall workflow composed of nodes and edges.
     - LangGraph supports:
       • StateGraph — the main graph type.
       • MessageGraph — a simpler graph for chat-based workflows.
       • Multi-agent graphs — sub-graphs for complex orchestration.

--- 3.2 Key Features ---

  • Cyclical Workflows — Unlike linear chains, LangGraph supports loops,
    branching, and recursion natively (e.g., agent-think-tool loops).

  • Persistence / Checkpointing — Save and restore graph state at any step.
    Supports in-memory, SQLite, and Postgres (PostgresSaver) backends.
    Enables human-in-the-loop, time travel, and fault tolerance.

  • Streaming — Stream outputs node-by-node or token-by-token for real-time
    UX. Supports streaming of both LLM tokens and state updates.

  • Memory Management
    - Short-term: state within a session (automatic).
    - Long-term: persistent memory via checkpointer + database.
    - Trim memory strategies to prevent context bloat (trim_memory_node).

  • Human-in-the-Loop — Pause execution, wait for human approval, then
    resume or modify state before continuing.

  • Visual Graph Generation — LangGraph can auto-generate Mermaid diagrams
    of your workflow for debugging and documentation.

4. HOW TO BUILD LANGGRAPH AGENTS (STEP-BY-STEP)
================================================================================

--- Step 1: Installation ---

  pip install langgraph langchain langchain-openai

  For the full ecosystem:
  pip install langgraph langchain langchain-openai langchain-community

--- Step 2: Define the State ---

  from typing import TypedDict, Annotated, Sequence
  from langgraph.graph.message import add_messages

  class AgentState(TypedDict):
      messages: Annotated[Sequence, add_messages]  # auto-merge messages

  Or with additional fields:
  class AgentState(TypedDict):
      messages: Annotated[Sequence, add_messages]
      next_step: str
      documents: list

--- Step 3: Define Nodes (Processing Steps) ---

  from langchain_openai import ChatOpenAI

  llm = ChatOpenAI(model="gpt-4o")

  def call_model(state: AgentState):
      response = llm.invoke(state["messages"])
      return {"messages": [response]}

  def tool_node(state: AgentState):
      # Tool execution logic here
      pass

--- Step 4: Build the Graph ---

  from langgraph.graph import StateGraph, START, END
  from langgraph.graph.graph import CompiledGraph

  # Initialize
  graph = StateGraph(AgentState)

  # Add nodes
  graph.add_node("agent", call_model)
  graph.add_node("tools", tool_node)

  # Add edges
  graph.add_edge(START, "agent")

  # Conditional edge — agent decides to use tools or respond
  def should_continue(state):
      last_message = state["messages"][-1]
      if last_message.tool_calls:
          return "tools"
      return END

  graph.add_conditional_edges("agent", should_continue, {
      "tools": "tools",
      END: END
  })
  graph.add_edge("tools", "agent")  # loop back to agent

  # Compile
  app = graph.compile()

--- Step 5: Add Memory / Checkpointing ---

  from langgraph.checkpoint.memory import MemorySaver

  memory = MemorySaver()
  app = graph.compile(checkpointer=memory)

  # For Postgres persistence:
  # pip install langgraph-checkpoint-postgres
  # from langgraph.checkpoint.postgres import PostgresSaver

--- Step 6: Run the Agent ---

  config = {"configurable": {"thread_id": "user-session-1"}}
  for chunk in app.stream(
      {"messages": [{"role": "user", "content": "What is LangGraph?"}]},
      config
  ):
      print(chunk)

--- Step 7: Visualize ---

  from IPython.display import Image, display
  display(Image(app.get_graph().draw_mermaid_png()))

5. BEST PRACTICES & RESOURCES
================================================================================

--- Best Practices ---

  • Keep state definitions minimal but extensible — add fields as needed.
  • Use conditional edges for agent decision loops (ReAct pattern).
  • Implement memory trimming to manage context window limits.
  • Use checkpointing for production deployments (Postgres > MemorySaver).
  • Design each node to do one thing well — single responsibility.

--- Learning Resources ---

  • Official LangGraph Docs: https://langchain-ai.github.io/langgraph/
  • DeepLearning.AI — "AI Agents in LangGraph" (free course by Harrison Chase)
  • freeCodeCamp — LangGraph Full Course for Beginners (YouTube, 3+ hours)
  • Tech With Tim — LangGraph Tutorial (YouTube)
  • DataCamp — LangGraph Agents Hands-On Tutorial
  • GitHub: LangGraph repo — https://github.com/langchain-ai/langgraph

--- Common Patterns ---

  1. ReAct Agent (Reason + Act): LLM -> Tool -> LLM -> Tool -> ... -> Respond
  2. Multi-Agent Supervisor: One agent delegates tasks to specialist agents
  3. RAG Agent: Retrieve documents -> Generate answer with context
  4. Human-in-the-Loop: Pause at critical decision points for human approval
  5. Reflection / Self-Critique: Agent generates, then reviews its own output

--- Key Libraries in the Ecosystem ---

  • langgraph — Core graph framework
  • langgraph-checkpoint — Persistence backends (memory, SQLite, Postgres)
  • langgraph-sdk — Deploy LangGraph agents as APIs
  • LangGraph Studio — Visual IDE for debugging LangGraph agents
  • LangSmith — Observability, tracing, and evaluation
